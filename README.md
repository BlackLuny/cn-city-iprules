# cn-city-iprules — 中国分省/分市 IP-CIDR 规则

基于 [metowolf/iplist](https://github.com/metowolf/iplist) 的 `cncity` 数据（IPIP 来源），
按省/地级市拆分，生成 **Surge** 与 **Clash** 两套规则文件，每个地区独立 URL，可直接在客户端用「文件/URL 导入」引用。

> 仅含 IPv4 CIDR。`100000` = 全国。

## URL 规律

把 `<code>` 换成下表的 6 位行政区划代码：

```
# Surge (RULE-SET, 内容形如 IP-CIDR,1.2.2.0/24,no-resolve)
https://raw.githubusercontent.com/BlackLuny/cn-city-iprules/main/surge/<code>.list
https://cdn.jsdelivr.net/gh/BlackLuny/cn-city-iprules@main/surge/<code>.list   # jsDelivr CDN，国内更快

# Clash (rule-provider, behavior: ipcidr, format: text, 纯 CIDR)
https://raw.githubusercontent.com/BlackLuny/cn-city-iprules/main/clash/<code>.txt
https://cdn.jsdelivr.net/gh/BlackLuny/cn-city-iprules@main/clash/<code>.txt
```

### Clash rule-provider 配置示例
```yaml
rule-providers:
  beijing:
    type: http
    behavior: ipcidr
    format: text
    url: https://cdn.jsdelivr.net/gh/BlackLuny/cn-city-iprules@main/clash/110000.txt
    path: ./ruleset/cn-110000.txt
    interval: 86400
rules:
  - RULE-SET,beijing,DIRECT
```

### Surge RULE-SET 示例
```ini
[Rule]
RULE-SET,https://cdn.jsdelivr.net/gh/BlackLuny/cn-city-iprules@main/surge/110000.list,DIRECT
```
